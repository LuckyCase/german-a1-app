## Текущее состояние

### Технологический стек
- **Backend:** Python 3.10+, python-telegram-bot (webhook)
- **Интерфейс:** Telegram Mini App (основной) + inline-режим (fallback)
- **БД:** PostgreSQL (asyncpg) в prod / SQLite для локальной разработки
- **Хостинг:** Render.com (Web Service)
- **Аудио:** gTTS (Google Text-to-Speech)

### Контент по уровням

| Раздел         | A1.1  | A1.2          | A2   |
|----------------|-------|---------------|------|
| Словарь        | ✅ 11 категорий (~262 слова) | ✅ 11 категорий | ❌ нет |
| Грамматика     | ✅ 8 тем (78 вопросов) | ✅ 8 тем | ❌ нет |
| Фразы          | ⚠️ 20 категорий (мало фраз в каждой) | ✅ 20 категорий | ❌ нет |

- ✅ Карточки с приоритетом ошибок (priority words)
- ✅ Грамматические тесты с теорией

- ✅ Диалоги с упражнениями
- ✅ Аудио произношение (gTTS)
- ✅ Трекер прогресса (слова, грамматика, фразы, диалоги, культура, упражнения)

- ✅ Система обратной связи со статусами

- ✅ Выбор уровня (A1.1 / A1.2) в настройках
- ❌ Spaced Repetition (SRS) — упрощённый подход без дат повторения
- ❌ Streak / геймификация

### Выявленные проблемы
1. **Тесты A1.1 слабые** — ~10 вопросов на тему мало, пользователь быстро запоминает варианты позиционно
2. **Фраз мало в категориях A1.1** — судя по README, на старте было 3 категории × 5 фраз; 20 категорий есть, но наполнение минимальное
3. **A1.2 не завершён** — culture и exercises пустые
4. **SRS отсутствует** — `correct_count >= 3` не даёт реального интервального повторения
5. **Нет мониторинга** — ошибки уходят только в лог Render
6. **Миграции БД через `init_db()`** — масштабируется плохо

## Фаза 1 — Закрытие долгов A1 (приоритет)
**Оценка: ~4–6 недель**

### 1.1 Усиление грамматических тестов A1.1

**Цель:** 20–25 вопросов на тему вместо ~10 (итого ~180–200 вопросов)

**Что добавить в каждую тему:**
- Вопросы на определение ошибки в предложении (`"Ich habe Hunger" vs "Ich bin Hunger"`)
- Вопросы на трансформацию (утверждение → вопрос)
- Более тонкие дистракторы (похожие формы, частые ошибки русскоязычных)

**Приоритетные темы для расширения:**
1. Артикли (der/die/das) — самая сложная для русскоязычных
2. Порядок слов (word_order) — сейчас всего 5 вопросов
3. Аккузатив — артикли меняются, нужно больше практики

**Структура JSON не меняется** — просто добавить объекты в массив `questions`.

### 1.2 Наполнение фраз A1.1

**Цель:** 10–15 фраз на категорию в каждой из 20 категорий

**Что добавить к каждой фразе:**
- Поле `register`: `"formal"` / `"informal"` / `"neutral"`
- Поле `response`: типичный ответ собеседника (DE + RU)

**Приоритетные категории (высокий практический спрос):**
- `greetings`, `cafe`, `shop`, `doctor`, `directions`, `station`, `restaurant`

### 1.3 Завершение A1.2

**Culture A1.2** — 5 новых тем (отличия от A1.1):
- Немецкие праздники (Weihnachten, Ostern, Oktoberfest)
- Система образования в Германии
- Деловой этикет (рукопожатие, обращение)
- Разделение мусора (Mülltrennung) — практически важно для переехавших
- Карманные деньги и чаевые

**Exercises A1.2** — 6–8 наборов, добавить темы:
- Глаголы с управлением
- Временные выражения
- Модальные глаголы в контексте

## Фаза 2 — Алгоритм и механики удержания
**Оценка: ~3–4 недели**

### 2.1 Spaced Repetition System (SRS)

**Текущая логика (упрощённая):**
```python
# Считается "выученным":
correct_count >= 3 AND wrong_count == 0
```

**Целевая логика (SRS):**

| Правильных ответов подряд | Следующий показ |
|--------------------------|-----------------|
| 0 (новое слово)          | сразу           |
| 1                        | +1 день         |
| 2                        | +3 дня          |
| 3                        | +7 дней         |
| 4                        | +14 дней        |
| 5+                       | +30 дней        |
| Ошибка                   | сброс до +1 дня |

**Изменения в БД:**
```sql
ALTER TABLE progress ADD COLUMN next_review_at TIMESTAMP;
ALTER TABLE progress ADD COLUMN streak INTEGER DEFAULT 0;
ALTER TABLE phrases_progress ADD COLUMN next_review_at TIMESTAMP;
ALTER TABLE phrases_progress ADD COLUMN streak INTEGER DEFAULT 0;
```

**Новая механика сессии:** при входе в раздел показывать сначала слова с `next_review_at <= NOW()`, затем новые. Это создаёт персональную очередь повторения каждый день.

### 2.2 Streak и геймификация

**Стрик:** серия дней подряд с активностью (≥1 слово/упражнение).
- Добавить поле `last_active_date` в таблицу `users`
- Вычислять стрик при каждой активности
- Отображать в `/progress` и в приветственном сообщении

**Простые ачивки (без новых таблиц — хранить в `users.achievements` как JSON):**

| Ачивка | Условие |
|--------|---------|
| 🌱 Первые шаги | 10 слов изучено |
| 🔥 Неделя подряд | Стрик 7 дней |
| 📚 Грамматик | Все тесты A1 пройдены |
| 💬 Болтун | 50 фраз изучено |
| 🏆 Мастер A1 | 80% прогресса по всем разделам A1 |

### 2.3 Новые типы упражнений

**Fill-in-the-blank (заполни пропуск):**
```json
{
  "type": "fill_blank",
  "sentence": "Ich ___ aus Deutschland.",
  "answer": "komme",
  "hint": "глагол kommen, форма для ich",
  "explanation": "Глагол kommen → ich komme (1-е лицо ед.ч.)"
}
```
Проверка: без учёта регистра, с подсказкой при одной ошибке (Levenshtein distance = 1).

**Matching (соединить пары):**
```json
{
  "type": "matching",
  "pairs": [
    {"left": "der Hund", "right": "собака"},
    {"left": "die Katze", "right": "кошка"},
    {"left": "das Pferd", "right": "лошадь"}
  ]
}
```

**Sentence Scramble (собери предложение):**
```json
{
  "type": "scramble",
  "words": ["Ich", "gehe", "heute", "in", "die", "Schule"],
  "answer": "Ich gehe heute in die Schule",
  "explanation": "Глагол стоит на втором месте"
}
```
Особенно полезно для отработки порядка слов (Wortstellung).

### 2.4 Диагностический тест при онбординге

При первом `/start` — предложить короткий тест (15 вопросов):
- 5 вопросов уровня A1 (базовые артикли, приветствия)
- 5 вопросов уровня A1+ (падежи, модальные)
- 5 вопросов уровня A2 (Perfekt, Dativ)

По результату:
- 0–5 правильных → рекомендовать A1.1
- 6–10 → рекомендовать A1.2
- 11–15 → рекомендовать A2 (когда появится)

Пользователь может согласиться или выбрать вручную.

## Фаза 3 — Уровень A2
**Оценка: 2–3 месяца**

### Структура A2 (аналогично A1)

```
data/A2/
├── 1/          ← A2.1
│   ├── vocabulary/
│   ├── grammar/
│   ├── phrases/
│   ├── dialogues/
│   ├── culture/
│   ├── exercises/
│   └── metadata.json
└── 2/          ← A2.2
    └── ...
```

### 3.1 Грамматика A2 (ключевые темы)

**A2.1:**
1. Perfekt (прошедшее время с haben/sein)
2. Dativ (дательный падеж)
3. Предлоги с Dativ (mit, bei, nach, von, zu, aus, seit)
4. Сравнительная степень прилагательных (groß → größer → am größten)
5. Придаточные с `weil` (порядок слов!)
6. Reflexivverben (возвратные глаголы: sich waschen, sich freuen)

**A2.2:**
1. Präteritum модальных (musste, wollte, konnte)
2. Придаточные с `dass` и `wenn`
3. Двойные артикли (kein → keinen, keine, kein)
4. Генитив (родительный — базовое знакомство)
5. Konjunktiv II (hätte, wäre, würde — вежливые просьбы)

### 3.2 Словарь A2 (~400 новых слов)

Новые категории для A2:
- Профессии и рабочее место
- Здоровье, симптомы, аптека
- Путешествия (паспорт, регистрация, задержка)
- Природа и погода (расширенно)
- СМИ: газеты, интернет, телевидение
- Квартира: аренда, договор, ремонт
- Банк и финансы (счёт, перевод, оплата)

### 3.3 Переход A1 → A2

После достижения ≥80% прогресса по A1 — показывать баннер с предложением перейти на A2. Итоговая карточка "Вы завершили A1!" со статистикой:
- Слов выучено / всего
- Грамматических тестов / всего
- Дней в стрике
- Дата начала и окончания

## Фаза 4 — Технический долг
**По мере возможности**

### 4.1 Миграции через Alembic

Текущий подход (`ALTER TABLE ADD COLUMN IF NOT EXISTS` в `init_db()`) работает, но при росте схемы становится неуправляемым. Внедрить Alembic:

```bash
pip install alembic
alembic init migrations
# Каждое изменение схемы — отдельная миграция-файл
```

### 4.2 Кеширование аудио

gTTS делает HTTP-запрос при каждом воспроизведении. Нужен файловый кеш:

```python
# Псевдокод
cache_path = f"audio_cache/{word_id}.mp3"
if not os.path.exists(cache_path):
    tts = gTTS(text=word, lang='de')
    tts.save(cache_path)
return cache_path
```

Для Render — хранить в `/tmp` (ephemeral) или подключить S3/R2 для персистентного хранения.

### 4.3 Мониторинг ошибок

Добавить Sentry (есть бесплатный tier):
```python
pip install sentry-sdk
import sentry_sdk
sentry_sdk.init(dsn="...", traces_sample_rate=0.1)
```

Или минимальный вариант: отправлять критические ошибки себе в Telegram через второй бот-токен.

### 4.4 Расширение админ-панели

Добавить в Flask (`web_server.py`) защищённые маршруты:
- `GET /admin/stats` — количество пользователей, DAU/MAU, популярные разделы
- `GET /admin/feedback` — просмотр и обновление статусов фидбека
- `POST /admin/feedback/<id>/status` — уже есть в БД (`feedback.status`), нужен UI
